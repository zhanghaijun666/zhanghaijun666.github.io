# Api加解密

## 设计方案

> 需求背景: 保证请求的数据安全，防止通过抓包获取数据。使用场景：官网数据、引流开发的游客访问资源等。

加密方案：采用RSA+AES混合加密方式。原因：非对称加密加密大文本的性能差，对称加密又容易破解。

### 实现方案

%%  G --> H[接收方]

%%  I -->|使用解密的AES密钥解密数据| J[解密后的数据]
%%  J --> K[显示给用户]
