kube-proxyдёҺiptables

жҲ‘д»¬зҹҘйҒ“ kube-proxy жҳҜ Kubernetes дёӯдёҖдёӘиҝҗиЎҢеңЁжҜҸдёӘиҠӮзӮ№дёҠзҡ„е®ҲжҠӨиҝӣзЁӢпјҢе®ғеҹәжң¬дёҠеҸҚжҳ дәҶйӣҶзҫӨдёӯе®ҡд№үзҡ„жңҚеҠЎе·Із»ҸеҜ№еҗҺз«Ҝ Pod иҙҹиҪҪеқҮиЎЎзҡ„и§„еҲҷз®ЎзҗҶгҖӮ

еҒҮи®ҫжҲ‘д»¬жңүеҮ дёӘ API еҫ®жңҚеҠЎзҡ„ Pods иҝҗиЎҢеңЁжҲ‘д»¬зҡ„йӣҶзҫӨдёӯпјҢиҝҷдәӣ Pods зҡ„еүҜжң¬йҖҡиҝҮдёҖдёӘ Service жңҚеҠЎжҡҙйңІпјҢеҪ“дёҖдёӘиҜ·жұӮеҲ°иҫҫ Service зҡ„иҷҡжӢҹ IP ж—¶пјҢеҰӮдҪ•е°ҶиҜ·жұӮиҪ¬еҸ‘еҲ°е…¶дёӯдёҖдёӘеә•еұӮ Podпјҹе…¶е®һе°ұжҳҜйҖҡиҝҮ kube-proxy еҲӣе»әзҡ„и§„еҲҷгҖӮ

# iptablesе’ҢIPvs

kube-proxy еҸҜд»ҘеңЁдёүз§ҚдёҚеҗҢзҡ„жЁЎејҸдёӢиҝҗиЎҢгҖӮ

  • iptables
  • IPvs
  • userspace(дёҚеҶҚжҺЁиҚҗ)

иҷҪз„¶ iptables жЁЎејҸеҜ№дәҺи®ёеӨҡйӣҶзҫӨе’Ңе·ҘдҪңиҙҹиҪҪжқҘиҜҙе®Ңе…ЁжІЎжңүй—®йўҳпјҢдҪҶеҪ“жңҚеҠЎж•°йҮҸеҫҲеӨҡж—¶пјҲи¶…иҝҮ1,000дёӘпјүпјҢipvs е°ұдјҡеҫҲжңүдјҳеҠҝдәҶпјҢз”ұдәҺ iptables и§„еҲҷжҳҜжҢүйЎәеәҸиҜ»еҸ–зҡ„пјҢеҰӮжһңйӣҶзҫӨдёӯеӯҳеңЁи®ёеӨҡжңҚеҠЎпјҢйӮЈд№Ҳе®ғзҡ„дҪҝз”ЁдјҡеҪұе“Қи·Ҝз”ұжҖ§иғҪгҖӮ

# еҲӣе»әзӨәдҫӢ

# еҲӣе»әдёҖдёӘеҹәдәҺ ghost й•ңеғҸзҡ„ DeploymentпјҲghost жҳҜдёҖдёӘе…Қиҙ№ејҖжәҗзҡ„еҚҡе®ўе№іеҸ°пјү
kubectl create deploy ghost --image=ghost --replicas=2
# дҪҝз”Ё NodePort зұ»еһӢзҡ„ Service жқҘжҡҙйңІ PodsгҖӮ
kubectl expose deploy/ghost --port 80 --target-port 2368 --type NodePort
# Service зҡ„зӣёе…ідҝЎжҒҜ
kubectl describe svc ghost
# жҲ‘д»¬д№ҹеҸҜд»ҘдҪҝз”Ёж ҮеҮҶзҡ„ kubectl get е‘Ҫд»ӨжқҘжҹҘиҜў Endpoints дҝЎжҒҜгҖӮ
kubectl get endpoints
